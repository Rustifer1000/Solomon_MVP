# Integrated Draft Snapshot

## Status

Archived integrated draft of the Solomon MVP evaluation specification.

This document predates the split into the active two-part structure:

- `docs/01_foundations_and_architecture.md`
- `docs/02_operations_and_evaluator_workflow.md`

It is preserved as a historical synthesis snapshot only.

It should **not** be treated as the primary active specification.

---

## Why this snapshot exists

At one point in drafting, the project had matured enough that it was useful to consolidate the working material into a single integrated document.

That integrated draft served several purposes:

- synthesis checkpoint
- structure test
- clarity review target
- preservation snapshot before splitting for canvas/repo usability

It was a necessary transitional artifact.

---

## Why it is no longer primary

The integrated draft is no longer the best working document because:

- it became too long for convenient editing
- it mixed architecture and operations more densely than ideal
- the split into Part I and Part II provides a cleaner reading path for developers

The active structure is now:

### Part I
Foundations and architecture:
- system definition
- design principles
- core/plugin boundary
- scoring model
- escalation model
- synthetic evaluation architecture

### Part II
Operations and evaluator workflow:
- benchmark scenarios
- evaluator score sheet
- evaluator instructions
- task list
- template families
- workflow-facing materials

---

## Historical value

This snapshot is useful for:

- preserving a single-file historical draft
- understanding the moment when the spec first became broadly developer-ready
- comparing later changes against an earlier integrated version

It is less useful for:
- ongoing editing
- developer onboarding
- normative interpretation

---
